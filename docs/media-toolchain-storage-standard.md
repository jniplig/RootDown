# Media Toolchain Storage Standard

## A. Overview

This standard defines one coherent storage model for the K6 media toolchain across DaVinci Resolve, OBS Studio, Blackmagic/ATEM, and Resolume.

The canonical root is:

`C:\Data`

The purpose of this model is consistency, recoverability, and reduced path sprawl. Media assets, application backups, project work, and app-managed folders should follow one predictable structure so the system remains understandable and supportable over time.

## B. Canonical Top-Level Mapping

### `C:\Data\30_MEDIA`

Use `C:\Data\30_MEDIA` for user-managed media storage:

- raw media
- edited or working media
- reusable creative assets
- exports and deliverables

### `C:\Data\50_SYSTEM`

Use `C:\Data\50_SYSTEM` for technical and recovery-oriented storage:

- application config exports
- backup and recovery folders
- project backup sets
- app support material that is not a primary creative asset

## C. Tool-by-Tool Standards

### 1. DaVinci Resolve

Use this standard:

- Media storage and cache-related root:
  `C:\Data\30_MEDIA\Video`
- Exports root:
  `C:\Data\30_MEDIA\Video\Exports\Resolve`
- Project backups:
  `C:\Data\50_SYSTEM\Config_Backups\Resolve_Project_Backups`

Operating rules:

- `_gallery` and `CacheClip` under the video root are app-managed.
- Exports are user-managed deliverables.
- Project backups are recovery assets.
- App-managed folders should not be manually reorganized unless the application workflow explicitly supports it.

Known validated state:

- DaVinci Resolve media storage is based under `C:\Data\30_MEDIA\Video`
- DaVinci Resolve exports are written under `C:\Data\30_MEDIA\Video\Exports\Resolve`
- DaVinci Resolve project backups are written under `C:\Data\50_SYSTEM\Config_Backups\Resolve_Project_Backups`

### 2. OBS Studio

Use this standard:

- Recordings:
  `C:\Data\30_MEDIA\Video\OBS_Recordings`
- Scene and profile exports or manual backups:
  `C:\Data\50_SYSTEM\Config_Backups\OBS`
- Reusable OBS assets such as overlays, stingers, graphics, and audio beds:
  `C:\Data\30_MEDIA\OBS\Assets`

Operating rules:

- Recordings are user-managed media.
- Exported scene collections and profiles are backup and rebuild assets.
- Active OBS app config paths may still live where OBS expects them unless explicitly exported or repointed.

### 3. Blackmagic / ATEM

Use this cautious standard:

- Backups, exported config, or autosaves that are safe to retain:
  `C:\Data\50_SYSTEM\Config_Backups\Blackmagic_ATEM`
- Media outputs generated by related tools:
  `C:\Data\30_MEDIA\Video\Blackmagic`

Operating rules:

- Folders such as `ATEM Autosave` and `Blackmagic Design` may be app-managed or workflow-sensitive.
- Repoint paths inside the application first where applicable.
- For risky folders, prefer copy-then-verify rather than immediate move.

### 4. Resolume

Use this standard:

- Resolume compositions and show files:
  `C:\Data\10_PROJECTS\Studio\Resolume_Shows`
- Reusable clips and visual assets:
  `C:\Data\30_MEDIA\Resolume\Assets`
- Optional exported backups or config snapshots:
  `C:\Data\50_SYSTEM\Config_Backups\Resolume`

Operating rules:

- Show-specific compositions belong with project work.
- Reusable clips belong in media storage.
- Backup and config exports belong in `50_SYSTEM`.
- App-managed folders should be handled cautiously.

## D. Recommended Canonical Subfolder Model

Use a subtree like this:

```text
C:\Data\30_MEDIA\
  Audio\
    Raw
    Edited
    Exports
  Video\
    Raw
    Edited
    Exports\
      Resolve
      OBS
      Blackmagic
  OBS\
    Assets
  Resolume\
    Assets

C:\Data\50_SYSTEM\
  Config_Backups\
    Resolve_Project_Backups
    OBS
    Blackmagic_ATEM
    Resolume

C:\Data\10_PROJECTS\
  Studio\
    Resolume_Shows
```

This model is not exhaustive, but it is the canonical direction for a consistent K6 media workflow.

## E. Rules for Repointing Application Paths

### Rules for Repointing Application Paths

- If a path is manually configured inside an application, repoint it in the app first.
- Verify that the application can read from and write to the new location.
- Only then move or archive the old folder.
- For uncertain app-managed folders, prefer copy-first instead of move-first.

## F. Naming Conventions for Media Outputs

Use the standard filename pattern:

`YYYY-MM-DD_Area_Item_Descriptor_v01.ext`

Approved `Area` values:

- `Teaching`
- `InsightEdu`
- `Studio`
- `Systems`
- `Personal`

Examples:

- `2026-04-07_Studio_Resolve-Test_H264_v01.mov`
- `2026-04-07_Studio_Lesson-Intro_1080p_v01.mp4`
- `2026-04-07_Studio_OBS-Overlay-Pack_Master_v01.zip`

## G. First Implementation Priorities

1. Resolve paths already validated.
2. Standardize OBS output and backup locations.
3. Review Blackmagic/ATEM folders with copy-first caution.
4. Standardize Resolume compositions and asset storage.
5. Keep generated reports and transient files out of the Git repo.

## H. Final Operating Principle

Keep media creation and storage under `30_MEDIA`, keep backup and recovery assets under `50_SYSTEM`, treat app-managed folders cautiously unless the application supports repointing, and keep project-specific creative work under the relevant project structure rather than mixing it into generic media storage.
